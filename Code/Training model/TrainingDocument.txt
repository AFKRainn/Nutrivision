NutriVision — training progress log
================================================================================
We record what we actually do in the notebook and on disk. TrainingStrategy.txt
is the plan; this file is the running log. When a "Next" item is finished, we move
it into "Done" and add a new Next.

goal: fine-tune on **all** datasets under Data/, not a single
export. Phase C full merge is required before the final train. Before committing to the
big merged dataset, we first ran a tiny smoke train on one dataset to confirm the
pipeline worked end-to-end (so we wouldn't discover a setup bug halfway through a long
run). The real fine-tune now runs from the standalone train.py script, not the notebook.

- Moved the final train out of the notebook into a standalone train.py. Running it as a
  plain process (not Jupyter) avoids the Windows dataloader-worker spawn deadlock/crash
  we kept hitting in the notebook. Settings: yolo12n.pt, merged data, epochs 35,
  patience 15, batch 16, workers 8, cache 'disk', cos_lr, close_mosaic 10, device 0.
- cache settings lesson: cache='ram' would have needed ~32 GB (with safety margin) and,
  combined with Windows worker spawn, blew up memory (MemoryError pickling the val
  cache to 8 workers). Switched to cache='disk', which fixed it.
- First full production train (train.py on the merged 181-class data): ran overnight and
  reached epoch 25/35 before crashing. Metrics at epoch 25: mAP50 0.605, mAP50-95 0.399,
  precision 0.658, recall 0.579. best.pt and last.pt saved under
  runs/nutrivision_merged_final/weights/.
- Crash cause: at epoch 26 close_mosaic rebuilt the train dataloader, which re-spawned
  8 Windows worker processes (each re-imports torch/CUDA). With the run already using
  GPU + disk cache + other apps, Windows ran out of page-file / virtual memory ->
  OSError WinError 1455 (paging file too small) while a worker loaded cufft64_12.dll.
- Decision: stop at epoch 25. mAP50 ~0.605 is a usable first detector, so we keep
  best.pt and move on rather than resume. Ideas to push accuracy later are recorded in
  Improvements.txt (kept local / git-ignored).
- Cleanup: removed the smoke-train and run1 cells and the in-notebook training (Phase D)
  from VisionModelTraim.ipynb. The notebook is now just the data pipeline
  (Phase A inventory -> Phase B harmonize -> Phase C merge). Deleted the leftover
  data.run1.yaml. Training lives in train.py.
